# Interview Formats — 面试形式参考

> 不同面试轮次有不同的形式、时长和评估标准。
> 供 question_generator.md 和 mock_interviewer.md 根据用户选择的轮次调整策略。

---

## Phone Screen / HR初筛

- **时长**: 15-30分钟
- **面试官**: HR / Recruiter
- **考察重点**: 基本背景匹配、沟通能力、薪资期望、签证状态
- **题目类型**: 自我介绍、为什么想来这家、职业规划、薪资范围
- **注意事项**: 这轮是筛选不是深度面试，保持简洁有条理
- **淘汰红线**: 经验严重不匹配、沟通困难、薪资差距太大

## Technical Phone Screen / 技术电话面

- **时长**: 45-60分钟
- **面试官**: Engineer (L4-L6)
- **考察重点**: 编码能力、问题分析、代码质量
- **题目类型**: 1-2道算法题（Medium为主）、简单系统设计（20min）
- **工具**: CoderPad / HackerRank / Google Docs
- **注意事项**: 边写边说、确认需求后再coding、考虑edge cases
- **时间分配**: 5min寒暄 + 35min coding + 5min questions

## Technical Onsite / 技术面（现场或视频）

- **时长**: 45-60分钟/轮
- **面试官**: Engineer (peer level or above)
- **子类型**:
### Coding Round
  - 1-2道算法题
  - 难度：Medium-Hard
  - 重点：正确性 + 复杂度分析 + edge cases + 代码质量

### System Design Round
  - 1个开放式设计题
  - 时间分配：5min需求确认 + 15min高层设计 + 15min深入某模块 + 5min扩展讨论
  - 重点：trade-off思维、scalability、实际经验
  - 画图工具：白板/Excalidraw/Google Docs

### Code Review / Debug Round（部分公司）
  - 阅读一段代码找bug或改进
  - 考察：code reading能力、工程素养
  - 时长：30-45分钟

## Behavioral Round / 行为面

- **时长**: 45-60分钟
- **面试官**: Engineering Manager / Director
- **考察重点**: 团队合作、冲突处理、领导力、价值观match
- **题目类型**: 全部STAR格式，6-8题
- **Amazon特别**: 每题对应2个Leadership Principles
- **常见框架**: STAR (Situation-Task-Action-Result), CARL (Context-Action-Result-Learning)
- **注意事项**: 准备6-8个不同的故事、覆盖不同能力维度

## Onsite Full Day / 全天面试

- **时长**: 4-6小时（含午餐）
- **轮次**: 4-5轮（2 coding + 1 system design + 1 behavioral + 1 culture fit）
- **体力准备**: 前一天早睡、带零食和水、午餐时也在被observe
- **心理准备**: 某一轮发挥不好不影响全局、保持consistent的精力
## Final Round / 终面

- **时长**: 30-60分钟
- **面试官**: VP / Director / Skip-level
- **考察重点**: 战略思维、文化fit、长期潜力、motivation
- **题目类型**: 
  - "Where do you see yourself in 5 years?"
  - "What would you do in your first 90 days?"
  - "Why this company / team / role?"
  - "What's the biggest impact you've made?"
- **注意事项**: 这轮不太考技术细节，更看big picture和人的气质

## Case Interview / 案例面试

- **适用于**: 咨询、PM、战略岗
- **时长**: 30-45分钟/case
- **结构**:
  1. 听题+确认（2min）
  2. 提出框架（3min）
  3. 分析数据（15min）
  4. 总结建议（5min）
- **评估标准**: 结构化思维、数据敏感度、商业直觉、沟通清晰度
- **常见类型**: Market sizing, Profitability, Market entry, M&A, Operations

## 面试形式 vs 准备策略映射

| 面试形式 | 核心准备 | 推荐练习量 |
|---------|---------|-----------|
| Phone Screen | 自我介绍 + 公司了解 | 1-2次mock |
| Technical Coding | LeetCode刷题 | 50-100题 |
| System Design | 看设计案例 + 画图练习 | 10-15个系统 |
| Behavioral | STAR故事准备 | 6-8个故事 |
| Full Day Onsite | 综合 + 体力管理 | 2-3次full mock |
| Final Round | 公司研究 + 职业规划 | 1-2次mock |
| Case Interview | 框架 + market sizing | 20-30个case |
